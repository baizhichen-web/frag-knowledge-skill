# 碎片知识 — Level 0

纯 Claude Code 驱动的知识管理工作流。零外部依赖，不需要 Obsidian、Anki 或任何 API。

核心理念：**AI 追问，人来总结。** AI 不替你想，而是帮你想到位。

## 触发条件（必读）

当用户消息匹配以下模式时，**必须**激活本 skill 并按流程执行，不能只做对话回复：

| 模式 | 流程 |
|------|------|
| "记一下xxx"、"帮我记一下xxx"、"记录一下xxx"、"捕获xxx" | → Flow 1: 捕获 |
| "翻一翻"、"看看卡片"、"今天看什么"、"复习一下" | → Flow 2: 翻阅 |
| "帮我审一下"、"哪些该整理了"、"知识库体检" | → Flow 3: 审阅 |
| "帮我完善xxx"、"更新xxx的反思"、"重新讨论xxx" | → Flow 2: 更新反思 |
| "做成卡片"、"帮我制卡"、"生成卡片" | → Flow 2: 制卡 |

**关键区分**：这不是一个对话 skill，而是一个工作流 skill。触发后必须执行具体的文件操作（创建目录、写入文件、读取文件、更新 JSON），不能只用文字回复。

## 存储结构

```
~/.frag-knowledge/
  inbox/              ← 新捕获（draft），按类型混放
  reviewing/          ← 正在讨论修改中的
  mature/             ← 归档的成熟笔记
    concepts/         ← 概念（方法论、原理、第一性原理）
    sources/          ← 来源（文章、演讲、论文的笔记）
    entities/         ← 实体（人、公司、工具、项目）
  review-state.json   ← FSRS 调度状态
  pending-cards.json  ← 待同步到 Anki 的卡片（Level 1+）
```

首次使用时自动创建：
```bash
mkdir -p ~/.frag-knowledge/{inbox,reviewing,mature/{concepts,sources,entities}}
```

**分类逻辑**：inbox/ 中不分类，AI 在 frontmatter 的 `type` 字段打标签。归档到 mature/ 时，AI 根据 type 自动放入对应子目录。用户说"这条放错地方了"可以随时调整。

## Flow 1: 捕获（"记一下xxx"）

### 触发词

- "帮我记一下..."
- "记一下..."
- "捕获..."
- "记录一下..."
- 任何包含"记"、"捕获"、"记录"且后面跟内容的消息

### 流程

1. **分类**（自动判断，不问用户，写入 frontmatter 的 `type` 字段）：
   - 包含 URL 或引用了具体文章/演讲/视频 → `source`
   - 含概念词（是、指、定义、原理、为什么、方法论）→ `concept`
   - 提到具体的人/公司/工具/项目 → `entity`
   - 其他 → `thought`（默认，归档时再细化）

2. **确认原文**：
   - 如果用户消息中已经包含了完整内容 → 直接用
   - 如果用户只给了概要、链接、或模糊描述 → 问用户："能把原文发给我吗？或者给我原文链接？"
   - **没有原文就不写文件**，先拿到原文再继续

3. **写入文件**（拿到原文后直接写，不等确认）：
   - 文件名：`{YYYY-MM-DD}-{slug}.md`，slug 从内容提取关键词
   - 写入 `~/.frag-knowledge/inbox/`
   - **必须原样保存用户提供的原文，不能改写、不能重新组织、不能删减**

4. **追问**（写完再问，不阻塞保存）：
   - 针对用户输入的内容，问一个苏格拉底式追问
   - 问题要具体，帮用户思考"为什么这对我重要"和"这和我已知的有什么关联"
   - 不要给答案或评价，只提问
   - 用中文，50 字以内
   - 用户可以回答，也可以说"跳过"

5. **共同反思**（用户回答追问后）：
   - 基于用户的回答，AI 提出自己的观察或角度，和用户一起讨论
   - 可以追问、可以补充、可以提出不同视角
   - 目标是通过对话一起得出一个有深度的反思
   - 讨论自然结束时，AI 问："我们讨论出来的这个想法，我帮你整理一下写进文件？"
   - **把讨论的成果（不是 AI 单方面的总结）写入 `## 反思` 字段**
   - 如果用户跳过追问，也跳过这步

6. **反馈**：一句话确认已保存，附带路径，并给出下一步建议：
   ```
   已保存到 ~/.frag-knowledge/inbox/2026-06-22-教育多样性与创新.md

   你还可以：
   - "翻一翻" → 看看你记过的其他内容
   - "帮我审一下" → 检查哪些笔记需要整理
   ```
   **只给一条建议，不要信息过载。优先建议"翻一翻"。**

### 文件格式

```markdown
---
type: {thought|concept|source}
date: {YYYY-MM-DD}
tags: [capture]
quality: draft
---

# {用户内容的前20字作为标题}

## 原始内容

{用户原始输入的内容，一字不改}

## 反思

{人和 AI 讨论后共同得出的反思。不是 AI 单方面的总结，也不是用户单方面的想法，而是对话的成果。没有讨论则留空}

## 追问记录

- Q: {追问}
- A: {用户的回答}
```

## Flow 2: 翻阅（"翻一翻"）

### 触发词

- "翻一翻"
- "看看卡片"
- "今天看什么"
- "复习一下"
- "看看笔记"

### 流程

1. **选卡**（从三个目录中按 FSRS 调度选取）：
   - 读取 `review-state.json`
   - 优先选 `next_due <= 今天` 的（到期的）
   - 到期的不够 5 条，用未翻阅过的补
   - 全部都看过了 → 告诉用户"今天全看完了，最早明天有 X 条到期"

2. **展示卡片列表**：
   ```
   今天 5 条：

     1. [draft] Transformer 的核心是自注意力 — 06-22
     2. [draft] 梯度消失是因为连乘 — 06-20
     3. [mature] Git rebase 是变基操作 — 06-15
     4. [needs-rework] 微服务的好处 — 06-10
     5. [draft] 协程比线程轻量 — 06-18

     想看哪条？或者说"再换一批"
   ```

3. **展开卡片**（用户说"看第 N 条"）：
   - 展示完整文件内容（原文 + 反思 + 追问记录）
   - 展示后提供操作：
     ```
     这条记得怎么样？
       忘了 / 模糊 / 记得住 / 很熟

     还可以：
       "帮我完善" → 重新讨论反思
       "做成卡片" → 从这条笔记生成复习卡片
       "这条成熟了" → 归档到 mature/ 对应子目录
       "这条要重做" → 标记待修改
     ```

4. **更新反思**（用户说"帮我完善"）：
   - 读取文件的 `## 原始内容` 和现有 `## 反思`
   - AI 基于原始内容发起新的讨论：
     ```
     这条的原文是关于教育多样性的。上次我们讨论的是 AI 如何增强课堂多样性。
     现在你对这个话题有什么新的想法吗？或者想从哪个角度深入？
     ```
   - 和 Flow 1 的"共同反思"一样：AI 提出观察，和用户讨论
   - 讨论结束后，**更新**文件的 `## 反思` 字段（不是追加，是替换）
   - 同时把新的讨论要点追加到 `## 追问记录`
   - quality 保持不变（不要自动改成 mature，让用户决定）

5. **直接更新**（不在翻阅中，用户主动说"帮我更新xxx的反思"）：
   - 搜索 inbox/, reviewing/, mature/ 中匹配的文件
   - 找到后进入和上面一样的"更新反思"流程
   - 找不到 → 列出最近的文件让用户选

6. **做成卡片**（用户说"做成卡片"或"帮我制卡"）：
   - 读取当前笔记的原文和反思
   - 用 Wozniak 20 Rules 生成 2-4 张卡片：
     - 最小信息原则：每张卡片只问一个知识点
     - 问题具体明确，不要问"什么是X的定义"
     - 答案简洁：1-3 句话
     - 问题本身要能独立理解
   - 卡片来源：
     - 原文中的关键论点 → 问答卡
     - 反思中的核心洞察 → 问答卡
     - 原文中的关键引用 → 摘录卡
   - 展示卡片预览：
     ```
     从这条笔记生成了 3 张卡片：

     卡1: 把最聪明的孩子聚在一起，丢失的是什么？
          → 教育多样性

     卡2: 诺贝尔化学奖获得者迈克尔莱利特认为顶尖科学家的时间花在哪里？
          → 99% 的时间在犯错

     卡3: AI 增强课堂多样性的核心思路是什么？
          → 不是模拟犯错，而是代表真正不同的思维路径

       "记住这些" → 保存到待同步文件
       "改一下第2张" → 修改卡片
       "再加一张" → 补充卡片
     ```
   - 用户确认后：
     - 检测 AnkiConnect（`curl -s http://localhost:8765`）
     - **在线** → 逐张添加到 Anki deck "frag-knowledge"
     - **不在线** → 保存到 `~/.frag-knowledge/pending-cards.json`

7. **批量制卡**（用户说"帮我把所有笔记都做成卡片"）：
   - 扫描 inbox/ + mature/ 下所有有 `## 反思` 的文件
   - 对每个文件生成卡片
   - 预览总数，确认后批量保存

### pending-cards.json 格式

```json
[
  {
    "question": "把最聪明的孩子聚在一起，丢失的是什么？",
    "answer": "教育多样性",
    "source": "inbox/2026-06-22-教育多样性与创新.md",
    "created": "2026-06-22"
  }
]
```

### AnkiConnect 调用（Level 1+）

检测到 AnkiConnect 在线时：
```bash
curl -s http://localhost:8765 -X POST -d '{
  "action": "addNote",
  "version": 6,
  "params": {
    "note": {
      "deckName": "frag-knowledge",
      "modelName": "问答题",
      "fields": {"正面": "{question}", "背面": "{answer}"},
      "tags": ["frag-knowledge"]
    }
  }
}'
```

### 评分（用户评价记忆程度）
   ```
   这条记得怎么样？
     忘了     → interval = 1 天
     模糊     → interval = Math.max(1, interval * 1.2)
     记得住   → interval = interval * 2.5
     很熟     → interval = interval * 4
   ```
   - 评分后更新 `review-state.json` 中的 `next_due`、`interval_days`、`review_count`
   - "忘了"和"模糊"同时标记 `quality: needs-rework`

### review-state.json 格式

```json
{
  "files": {
    "inbox/2026-06-22-attention-is-all-you-need.md": {
      "last_reviewed": "2026-06-22",
      "review_count": 3,
      "interval_days": 4,
      "next_due": "2026-06-26"
    }
  }
}
```

首次翻阅时如果文件不存在，自动创建空结构。

### 选卡算法

```
1. 扫描 inbox/, reviewing/, mature/ 下所有 .md 文件
2. 对每个文件：
   - 在 review-state.json 中查找
   - 如果没有记录 → 视为新卡，优先级最高
   - 如果 next_due <= 今天 → 到期卡，优先级高
   - 如果 next_due > 今天 → 未到期，跳过
3. 按优先级排序，取前 5 条
4. 如果不足 5 条，从未翻阅过的新卡中补
```

## Flow 3: 审阅（"帮我审一下"）

### 触发词

- "帮我审一下"
- "哪些该整理了"
- "知识库体检"
- "哪些笔记写得不好"
- "回炉"

### 流程

1. **扫描**（自动分析 inbox/ 下所有文件）：
   - 按规则标记问题：
     - 没有 `## 反思` 或反思为空 → "缺反思"
     - 只有追问记录没有正文 → "碎片未整理"
     - 超过 60 天没被翻阅 → "很久没看了"
     - `quality: draft` 且超过 30 天 → "待升级"

2. **输出报告**：
   ```
   扫描了 23 条笔记：

     🔴 5 条缺反思（没有讨论过）
     🟡 3 条碎片未整理
     🟡 2 条超过 60 天没看
     🔵 4 条仍是 draft 超过 30 天

     建议从这 3 条开始（最容易改进）：
       1. "微服务的好处" — 只有 20 字，没有总结
       2. "协程比线程轻量" — 只有追问记录
       3. "梯度消失" — 60 天没看了

     要开始回炉吗？选一条，我帮你重新梳理。
   ```

3. **回炉对话**（用户选中一条后）：
   - 读取文件的原文和现有反思
   - 引导用户重新理解这条知识：
     ```
     这条是你 6 月 10 号记的："微服务的好处"
     上次我们讨论过 xxx。现在你对这个有什么新的理解吗？
     ```
   - 进入和 Flow 2"更新反思"一样的讨论流程
   - 讨论结束后，更新文件的 `## 反思`
   - quality 改为 mature，按 type 移到 mature/{concepts|sources|entities}/
   - 更新 review-state.json

## 目录流转

```
捕获 → inbox/（draft, frontmatter 带 type 标签）→ 反馈时建议"翻一翻"
         │
    翻阅展开 → 评分（忘了/模糊/记得住/很熟）
         │
    ├─ "忘了/模糊" → quality: needs-rework
    ├─ "帮我完善" → 讨论更新反思 → 仍在当前位置
    ├─ "这条成熟了" → quality: mature → 按 type 移到 mature/{concepts|sources|entities}/
    └─ "这条要重做" → 移到 reviewing/
         │
    审阅回炉 → 讨论更新反思 → quality: mature → 按 type 移到 mature/
         │
    主动说"帮我更新xxx的反思" → 搜索文件 → 讨论更新
```

归档时 AI 读取 frontmatter 的 `type` 字段，自动放入 mature/ 对应子目录。
用户说"这条放错地方了" → 移动到正确子目录。

mature/ 不是死档案。到期了照样翻出来，只是间隔长（初始 90 天）。

## 自动信号（审阅时额外检查）

只保留两个自动信号，其余靠人和 AI 对话判断：
- 超过 60 天没被翻阅过的 → 提醒"这条很久没看了"
- `quality: draft` 且超过 30 天 → 提醒"这条还没反思过"

## 环境升级提示

每次执行完操作后，检测环境并给出**一条**引导（不要一次全说）：

| 检测项 | 引导 |
|--------|------|
| 没有 Obsidian vault | "内容保存在 `~/.frag-knowledge/`。安装 Obsidian 可以在手机上随时查看" |
| 没有 Anki | "说'做成卡片'可以生成 Anki 卡片，需要安装 Anki + AnkiConnect" |
| 有 Anki 但 AnkiConnect 不在线 | "Anki 已安装但 AnkiConnect 插件未启用" |
| 一切都配好了 | 不提示 |

**关键：每次只给一条引导，不要信息过载。**
