# AI 进课堂：通用思维镜头

## 框架结构

```
输入（课堂上下文）→ 思维镜头（Lens）→ 输出（激发性问题/观察）
```

AI 进课堂不是当"好学生"或"犯错的学生"，而是当一个**来自不同维度的闯入者**——用人类学生不会有的思维方式，激发课堂的创新活力。

## 6 个通用镜头

### 1. 跨界迁移
**公式**：如果这个设计用在 [完全不同的领域]？
- "这个杯子如果给盲人用呢？"
- "这个APP如果给宇航员在太空中用呢？"

### 2. 时间穿越
**公式**：如果 [过去/未来] 的人遇到这个问题？
- "100年后的人回头看这个设计，会觉得哪里可笑？"
- "如果1920年代的设计师拿到这个brief，他会怎么做？"

### 3. 极限条件
**公式**：如果 [资源/用户/环境] 极端受限？
- "如果这个产品只能用一种材料？"
- "如果预算只有10块钱？"

### 4. 反转目标
**公式**：如果目标不是 [便利/美观]，而是 [相反]？
- "如果这个APP的目标是让用户少用它？"
- "如果这个椅子的目标是让人不舒服？"

### 5. 拆解假设
**公式**：你这个方案隐含了一个假设：[X]。如果这个假设不成立？
- "你假设用户站着用。如果用户是躺着的呢？"
- "你假设这个产品在国内卖。如果用户在非洲呢？"

### 6. 跨感官
**公式**：如果用户不是用 [视觉]，而是用 [其他感官]？
- "如果这个产品只能用触觉来操作？"
- "如果这个界面没有颜色，只有形状？"

## 介入时机

| 时机 | 推荐镜头 | 原因 |
|------|---------|------|
| 学生方案初步成型 | 拆解假设 | 发现隐含前提 |
| 讨论陷入同质化 | 跨界迁移 / 反转目标 | 打破思维惯性 |
| 方案快定稿 | 极限条件 | 压测脆弱点 |
| 课间/课后 | 时间穿越 | 留悬念，延续思考 |
| 设计过度依赖视觉 | 跨感官 | 拓展感知维度 |

## 使用原则

1. 锚定当前话题 — 和正在讨论的内容有连接点
2. 具体到可以想象 — 学生能在脑子里"看到"的场景
3. 没有显而易见的答案 — 不是 Google 能找到的
4. 激活已有经验 — "你其实知道，但从没往这个方向想过"
5. 一次只用一个镜头 — 不要同时扔多个问题
6. 不要给答案 — 问题本身就是价值
